## Plugins

Track page views, referrers and devices on your Kirby 3 website. [Kirby 3 Simplestats](https://github.com/Daandelange/kirby3-simplestats) provides a minimal solution for self-hosted and non-intrusive visitor analytics.

Cache bust your assets without using query strings with a little help from the [Kirby Hashed Assets](https://github.com/johannschopplich/kirby-hashed-assets) plugin.

[Kirby Locked Pages](https://github.com/johannschopplich/kirby-locked-pages): Access-protect pages in your Kirby installation with a login

<small>Always test third-party plugins thoroughly before using them in production. Note that we do not test plugins we list here.</small>

## Cookbook recipes

Do you want to get better at debugging your PHP code and leave `var_dump()` behind? Then check out our new step-by-step recipe [how to get Xdebug up and running](https://getkirby.com/docs/cookbook/php/php-debugging-with-xdebug) together with VS Code.

If you want to store content in the new Blocks field from the frontend, the [Update blocks programmatically](https://getkirby.com/docs/quicktips/update-blocks-programmatically) recipe will come in handy.

Delivering your assets and files using a Content Delivery Network (CDN) made easy with [Kirby loves CDN](https://getkirby.com/docs/cookbook/extensions/kirby-loves-cdn).

[Keep your code dry](https://getkirby.com/docs/quicktips/keep_your_code_dry) has some tips how to increase your code's performance and circumvent typical traps.

## Privacy

[GitHub has removed all cookie banners from their websites](https://github.blog/2020-12-17-no-cookie-for-you/). Let's hope that others will follow suit, because they are generally a pain in the neck worsened by the dark patterns often used in these banners.

## PHP

"PHP doesn't have to be terrible": [Why Vimeo is still happily using PHP in 2021](https://medium.com/vimeo-engineering-blog/its-not-legacy-code-it-s-php-1f0ee0462580).

[PHP-blurhash](https://github.com/kornrunner/php-blurhash) is a PHP implementation of the [Blurhash](https://github.com/woltapp/blurhash) library that can be used to create image placeholders.

[Clockwork](https://underground.works/clockwork/) is a development tool that allows you to debug PHP right in the browser.

## Frontend

Elaina Natario [explains the semantic differences between the `alt` attribute and the `figcaption` tag](https://thoughtbot.com/blog/alt-vs-figcaption) and how they can coexist.

Sarah Higley wrote about [table and grid semantics](https://sarahmhigley.com/writing/grids-part2/).

Create a SPA with Vue 3 and Kirby based on Johann Schopplich's [Kirby Vue Starterkit](https://github.com/johannschopplich/kirby-vue3-starterkit): SEO-friendly, automatic routing, multi-lang-ready and more.

[Is progressive enhancement dead yet?](https://briefs.video/videos/is-progressive-enhancement-dead-yet/) A well crafted, entertaining Heydon Pickering video reminding us of the importance of progressive enhancement.

Christian Heilmann shows how to use browser dev tools to [find potential accessibility issues](https://christianheilmann.com/2021/01/11/seven-ways-to-test-for-accessibility-of-your-web-site-with-browser-developer-tools/) in websites.

## This & That

[Learn how to use Unix pipes](https://uwe-gehring.de/en/blog/arbeiten-mit-unix-pipes), fittingly explained with an example how to find all `t()` helper occurrences in your Kirby project.

[LinkAce](https://www.linkace.org/) is a self-hosted bookmark archive to store and organize links of your favorite websites.
